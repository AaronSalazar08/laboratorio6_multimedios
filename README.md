En el dashboard: **SQL Editor в†’ New query**, pega este SQL completo y ejecГєtalo:

```sql
-- =========================================================
-- Schema de Taskboard
-- =========================================================

create table public.tasks (
  id          bigint generated by default as identity primary key,
  title       text   not null check (char_length(title) <= 120),
  description text   default '',
  priority    text   not null default 'normal'
              check (priority in ('urgent','high','normal','low')),
  due_date    timestamptz,
  status      text   not null default 'pending'
              check (status in ('pending','progress','done')),
  created_at  timestamptz not null default now(),
  updated_at  timestamptz not null default now()
);

create table public.tags (
  id         bigint generated by default as identity primary key,
  name       text   not null unique check (char_length(name) <= 32),
  color      text   not null check (color ~ '^#[0-9a-fA-F]{6}$'),
  created_at timestamptz not null default now()
);

create table public.task_tags (
  task_id bigint not null references public.tasks(id) on delete cascade,
  tag_id  bigint not null references public.tags(id)  on delete cascade,
  primary key (task_id, tag_id)
);

-- Mantener updated_at fresco automГЎticamente.
create or replace function public.set_updated_at() returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger tasks_set_updated_at
  before update on public.tasks
  for each row execute function public.set_updated_at();

-- RLS: uso personal abierto.
-- TODO: cuando agregue auth, restringir por user_id = auth.uid().
alter table public.tasks      enable row level security;
alter table public.tags       enable row level security;
alter table public.task_tags  enable row level security;

create policy "personal_use_open" on public.tasks      for all using (true) with check (true);
create policy "personal_use_open" on public.tags       for all using (true) with check (true);
create policy "personal_use_open" on public.task_tags  for all using (true) with check (true);

-- Realtime sobre las tres tablas.
alter publication supabase_realtime add table public.tasks;
alter publication supabase_realtime add table public.tags;
alter publication supabase_realtime add table public.task_tags;
```

## Accesibilidad (WCAG 2.1 AA)

Decisiones implementadas:

- **NavegaciГіn 100% por teclado**: cards traen botones `в†ђ` / `в†’` para mover entre estados sin drag, con `disabled` en los extremos. Drag & drop sigue disponible para mouse.
- **Foco visible**: `:focus-visible` global con `outline: 2px solid var(--color-accent)` + `outline-offset: 2px`.
- **Labels accesibles**: todos los inputs sin label visible (search, tag input, color picker) usan `<label class="visually-hidden">` (clip-path tГ©cnica) en lugar de `aria-label` cuando es viable.
- **Estructura semГЎntica**: `<section>` con `aria-labelledby`, `<dialog>` con `aria-labelledby`, listas asociadas a sus encabezados.
- **Anuncios dinГЎmicos**: badges de columnas con `aria-live="polite"` y `aria-label` descriptivo (singular/plural correcto: "1 tarea pendiente" vs "3 tareas pendientes"). Errores de form con `role="alert"`.
- **`aria-invalid` sincronizado** en el input de tГ­tulo cuando se intenta submit vacГ­o; se limpia al empezar a escribir.
- **Contraste**: todos los colores muted en dark mode pasan AA holgadamente (в‰Ґ4.5:1, en prГЎctica ~9.6:1 para texto secundario).
- **Sin trampas de foco**: el `<dialog>` nativo gestiona focus trap automГЎtico.
- **`prefers-reduced-motion`**: animaciones de toast y transiciones se desactivan si el usuario lo pidiГі.

Lighthouse Accessibility esperado: **в‰Ґ 95**.

## Comportamiento offline

- **Reads**: si Supabase estГЎ caГ­do o credenciales mal configuradas, la app lee del cache localStorage (`taskboard_cache_v1`) y muestra banner amarillo de "modo solo lectura".
- **Writes**: en esta versiГіn las escrituras NO se guardan offline. Si Supabase no responde al guardar, sale un toast de error y la UI no cambia.
- **Realtime**: si pierdes conexiГіn, la suscripciГіn se cae silenciosamente. Al volver online (`window.online`), refresca solo.
